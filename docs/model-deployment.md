# 4.1.4 模型端实现

本项目的核心手语翻译模型 SOKE (Signs as Tokens) 在 AutoDL 平台上进行推理服务部署，模型实现主要基于 PyTorch 深度学习框架，使用的多语言手语生成模型采用开源社区公开的权重（ICCV 2025 论文 "Signs as Tokens" 官方实现）。

关于手语翻译模型使用的数据集，包括 How2Sign、CSL-Daily、Phoenix-2014T 等连续手语数据集，已在第二部分做相关介绍，在此不再赘述。

# 4.1.5 系统部署

AI 应用的部署一大难点在于 GPU 服务器的获取。由于 3D 手语生成模型推理需要较大的显存（约 10GB），且 SOKE 模型依赖复杂的 3D 渲染环境（PyRender、SMPL-X 等），对部署环境有较高要求。对于个人学生开发者而言，高昂的 GPU 服务器租用费用（1 个月近千元）同样很难接受。

基于此，团队选择了 AutoDL GPU 算力云平台作为部署方案。AutoDL 提供了预配置好 CUDA、PyTorch 等深度学习环境的容器，大大降低了部署复杂度。通过与平台方沟通，申请到了对外开放的端口映射服务，使得模型推理服务可以通过公网访问。

项目整体的部署方案采用模型推理服务 + 前端分离部署。我们为模型端基于 FastAPI 编写了 RESTful API 接口，提供文本到手语视频的生成能力；前端采用纯 HTML/JavaScript 实现，可直接部署在任意静态服务器或 CDN 上。当前，前端页面托管于本地开发环境，通过公网地址访问后端 API 进行测试。

模型推理服务监听 6008 端口，通过 AutoDL 平台的端口映射功能，将内网服务暴露到公网 HTTPS 地址，实现前端与后端的跨域通信。
